📖 사용 가이드

STEP 1

사전 설치

🔧 ffmpeg 설치 (필수 — m4a/wav 등 오디오 파일 처리)
# 방법 1: winget (Windows 10/11 권장)
winget install ffmpeg

# 방법 2: 수동 설치
# https://ffmpeg.org/download.html → Windows 빌드 다운로드
# 압축 해제 후 bin 폴더를 시스템 PATH에 추가

# 설치 확인
ffmpeg -version
📦 Python 패키지 설치
pip install -r requirements.txt
STEP 2

오디오 파일 배치

프로젝트 폴더 안의 audio/ 폴더에 명령어명 폴더를 만들고 녹음 파일을 넣어주세요. 파일은 최대 10개를 권장합니다.

📂 폴더 구조 예시
audio/
  ├── Check tomorrow's schedule/
  │     ├── Calendar tomorrow 1.m4a
  │     ├── Calendar tomorrow 2.m4a
  │     └── ... (최대 10개)
  └── What's my schedule today/
        ├── Calendar today 1.wav
        └── ...
✅ 지원 오디오 포맷
.m4a.wav.mp3.flac .aac.ogg.wma.mp4 .opus.aiff.amr.3gp

※ ffmpeg가 설치되어 있으면 위 포맷 모두 지원됩니다.

STEP 3

분석 실행

▶ 서버 실행
python app.py
🌐 브라우저 접속
http://localhost:5000

브라우저에서 ▶ 분석 시작 버튼을 클릭하면 audio/ 폴더의 모든 파일이 자동으로 분석됩니다. 파일이 1개뿐이어도 분석됩니다.

구간시작끝설명
T0사용자 음성 끝음성인식음 시작기기가 음성을 인식하기까지의 대기 시간
T1음성인식음 시작첫 번째 중간음 (없으면 최종응답)인식 후 첫 반응까지의 시간
T2첫 번째 중간음두 번째 중간음 (없으면 0)중간음 사이 처리 시간
T3마지막 유효 중간음최종응답 시작 (없으면 0)마지막 중간음 후 최종 응답까지
E2ET1 + T2 + T3최종 응답속도 (T0 미포함)

TIP

감지 정확도 조정 (config.py)

자동 감지가 부정확할 경우 config.py에서 아래 값을 조정하세요.

파라미터기본값설명
THRESHOLD_OFFSET_DB15높이면 더 큰 소리만 감지 (잡음이 많을 때 올리세요)
MIN_SEGMENT_DURATION0.15s이보다 짧은 구간은 잡음으로 무시
MAX_MERGE_GAP0.08s이 간격 이하의 침묵은 같은 구간으로 병합
